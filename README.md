# DB-project-library
Индивидуальный проект по БД
Постановка задачи:
Спроектировать и реализовать базу данных "Библиотека" для учета книг, читателей и бронирований книг в читальном зале 
Задание:
1. Создание БД и таблиц:
Создайте базу данных library со следующими таблицами.
   ◦ books – книги (издания). Для каждой книги должна сохраняться информация о количестве доступных экземпляров (available_copies), название, идентификатор издания.
   ◦ readers – читатели. Для каждого читателя в библиотеке должен сохраняться адрес электронной почты и телефон. Корректность email и номера телефона должна обеспечиваться на уровне БД (реализовать необходимые ограничения). Также необходимо сохранять статус читателя (is_active)
   ◦ book_rentals- журнал выдач\возвратов. В журнале должна сохраняться информация о типе выдачи (rental_type - 'home', 'reading_room'), дате выдачи, дате предполагаемого возврата и дате фактического возврата. Данная таблица должна быть секционирована по диапазонам на основе даты выдачи. Необходимо создать секции для хранения журнальных записей за 2024 и 2025 года (book_rentals_2024 и book_rentals_2025). 
   ◦ reading_room_bookings – журнал бронирований читального зала. В журнале должна сохраняться информация о том, кто из читателей какую книгу забронировал и на какой период (time_from, time_to). Допускается бронирование только на будущее время. Также необходимо сохранять информацию о статусе брони (status - 'active', 'completed', 'cancelled'). Реализуйте в таблице ограничение, для предотвращения одновременных АКТИВНЫХ бронирований одной книги на пересекающиеся периоды времени
   ◦ fines – штрафы. Данная таблица должна содержать информацию о номере выдачи, сумме штрафа, причине штрафа (просрочка возврата на % дней), а также сроке оплаты штрафа (не более 30 дней с текущей даты). 
Определите первичные ключи, внешние ключи, ограничения NOT NULL и CHECK.

2. Индексы:
   • Создайте не менее трех специализированных индексов с обоснованием: для чего необходим данный индекс и какие запросы выиграют от его использования.
   
3. Процедурные объекты:
   • Напишите функцию get_reader_stats(reader_id INT) для получения статистики по выбранному читателю (входной параметр). Функция должна возвращать таблицу с информацией о количестве текущих выдач, просроченных выдач, активных бронирований в читальном зале, а также общее количество нарушений (просрочки + отмененные брони).
   • Реализуйте триггер trg_check_book_availability_before_rent проверки доступности книги перед ее выдачей (перед добавлением записи в журнал выдач). Триггер должен проверять наличие доступных экземпляров книги, а также количество книг на руках у читателя. Операция выдачи должна быть отменена если у читателя на руках более 5 книг или нет доступных экземпляров. Если все проверки пройдены, триггер должен уменьшить количество доступных экземпляров книги.
Требования к процедурным объектам:
  Для функций:
    • Должны иметь корректный уровень изменчивости (IMMUTABLE, STABLE, VOLATILE)
    • Должны содержать валидацию входных параметров
    • Должны обрабатывать исключительные ситуации
    • Должны возвращать осмысленные сообщения об ошибках
  Для процедур:
    • Должны использовать блоки BEGIN...EXCEPTION...END для обработки ошибок
    • Должны проверять бизнес-правила перед выполнением операций
    • Должны обеспечивать целостность данных при ошибках
   
4. Управление транзакциями:
  • Смоделируйте ситуацию "гонки данных": два библиотекаря одновременно пытаются выдать последний экземпляр популярной книги. Напишите транзакцию для операции выдачи книги, которая гарантированно предотвратит двойную выдачу последнего экземпляра.
  • Обоснуйте выбор уровня изоляции транзакции. Продемонстрируйте, что выбранный уровень изоляции решает проблему.

5. Анализ производительности:
  • Напишите запрос, который показывает 10 самых популярных книг за последний год.
  • Проведите анализ плана выполнения этого запроса с помощью EXPLAIN (ANALYZE, BUFFERS).
  • На основе анализа предложите и реализуйте меры по оптимизации. Предоставьте план выполнения до и после оптимизации.
  • Проанализируйте эффективность секционирования таблицы book_rentals и предложите возможные улучшения структуры секционирования
